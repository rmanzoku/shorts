# Research Workflow

ネタについてWeb調査し、調査メモを作成する: $ARGUMENTS

> 全体フロー: ネタ提供 → **`/research`** → 調査レビュー → `/conte` → コンテレビュー → `/generate` → `/improve` → アップロード

## Step 1: Web調査

WebSearch で関連情報を収集する:

- 日本語ニュース記事を3本以上検索（年号を含めて検索）
- 一次情報源（公式発表、統計データ等）を優先する
- 矛盾する情報がある場合は複数ソースで裏付けをとる

```
WebSearch: "<トピック> 2025" / "<トピック> 最新"
```

## Step 2: 情報整理・ファクトチェック

収集した情報を整理:

- 事実と意見を明確に分離する
- 数字・日付・固有名詞は元ソースと照合する
- 偏向・扇動的な表現を含まないよう注意する
- 出典URLを記録する

## Step 3: 調査メモ作成

連番を確認し、`contes/NNN_slug.research.md` に出力する:

```bash
ls contes/   # 既存の連番を確認 → 次の番号を使う
# 例: 001_team-mirai-rise.md が最後なら → 002_next-topic.research.md
```

以下のフォーマットで調査メモを作成する:

```markdown
# <トピック名>

## 概要
<トピックの簡潔な説明（2〜3文）>

## 主要な事実
- <事実1（数字・日付を含める）>
- <事実2>
- <事実3>
...

## 背景・文脈
<トピックの背景情報、なぜ今重要なのか>

## 多角的な視点
- **賛成/肯定的な見方**: ...
- **反対/批判的な見方**: ...
- **中立的な分析**: ...

## 動画の切り口候補
- <切り口1: フックになりそうな事実や疑問>
- <切り口2>
- <切り口3>

## 出典
- [記事タイトル1](URL1)
- [記事タイトル2](URL2)
- [記事タイトル3](URL3)
```

### 調査メモのポイント

- **数字を集める**: 具体的な数値は動画の説得力とフックに直結する
- **「なぜ？」を掘り下げる**: 視聴者の疑問に答える材料を揃える
- **切り口候補**: コンテ作成時にフックとして使える事実・疑問を提案する

## Step 4: ユーザーにレビュー依頼

調査メモをユーザーに提示し、以下を確認してもらう:

- 事実関係に誤りがないか
- 取り上げるべき視点が抜けていないか
- 動画の切り口はどれが良いか

レビュー完了後、`/conte contes/NNN_slug.research.md` でコンテ作成に進む。
