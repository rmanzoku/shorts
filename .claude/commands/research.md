# Research & Conte Creation Workflow

ネタについて調査し、動画用コンテ（Markdown）を作成する: $ARGUMENTS

## Step 1: Web調査

WebSearch で関連情報を収集する:

- 日本語ニュース記事を3本以上検索（年号を含めて検索）
- 一次情報源（公式発表、統計データ等）を優先する
- 矛盾する情報がある場合は複数ソースで裏付けをとる

```
WebSearch: "<トピック> 2025" / "<トピック> 最新"
```

## Step 2: 情報整理・ファクトチェック

収集した情報を整理:

- 事実と意見を明確に分離する
- 数字・日付・固有名詞は元ソースと照合する
- 偏向・扇動的な表現を含まないよう注意する
- 出典URLを記録する

## Step 3: コンテ作成

連番を確認し、`contes/NNN_slug.md` に出力する:

```bash
ls contes/   # 既存の連番を確認 → 次の番号を使う
# 例: 001_soka-gakkai.md が最後なら → 002_next-topic.md
```

以下のフォーマットで `.md` ファイルを生成する:

```markdown
# <タイトル（興味を引く疑問形推奨）>

## シーン 1
**映像**: <映像の説明（AI画像生成の指示になる）>
**ナレーション**: <TTS読み上げテキスト>

## シーン 2
**映像**: <映像の説明>
**ナレーション**: <TTS読み上げテキスト>

...（3〜6シーン）
```

### 台本ガイドライン

- です・ます調で統一
- 読点は1文に2つ以内
- ナレーション合計: 90秒以内（約525文字）
- シーン数: 3〜6
- 偏向・扇動的表現の禁止
- 映像指示は具体的に（「街の風景」ではなく「渋谷スクランブル交差点の夜景、ネオンが反射する雨上がりの路面」）

### 映像指示の注意点

- テキスト・文字・ロゴを含む映像指示は避ける（AI画像で文字化けするため）
- 人物の顔アップは避ける（AI画像の弱点）
- 風景・建物・物体・抽象的なイメージを推奨

## Step 4: ユーザーにレビュー依頼

作成したコンテをユーザーに提示し、以下を確認してもらう:

- 事実関係に誤りがないか
- 構成・シーン分割は適切か
- 映像のイメージは意図通りか
- ナレーションのトーンは適切か

修正が必要な場合はコンテを更新する。
レビュー完了後、`/generate contes/NNN_slug.md` で動画生成に進む。
出力先は `output/NNN_slug.mp4` を推奨。
