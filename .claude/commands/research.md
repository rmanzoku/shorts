# Research Workflow

ネタについてWeb調査し、調査メモを作成する: $ARGUMENTS

> 全体フロー: ネタ提供 → **`/research`** → 調査レビュー → `/conte` → コンテレビュー → `/generate` → `/improve` → アップロード

## Step 0: トピック評価

動画にする前にトピックのエンゲージメントポテンシャルを評価する:

| 基準 | 1点（弱い/低リスク） | 3点（普通/中リスク） | 5点（強い/高リスク） |
|------|------------|------------|------------|
| 対象の広さ | 特定の人物/組織 | 政策分野 | 生活に直結（お金・仕事・安全） |
| 感情的インパクト | 中立的事実のみ | 軽い驚き | 衝撃・怒り・希望 |
| 議論性 | 一方的な事実 | やや意見が分かれる | 賛否が割れる（コメントが来る） |
| 抑制リスク | 経済の仕組み・技術・国際比較 | 特定の政治家名・選挙結果 | 宗教団体名・民族・差別関連 |

**進行条件:**
- エンゲージメント（対象の広さ + 感情 + 議論）合計 ≥ 9点
- 抑制リスク ≤ 3

**抑制リスク = 5 の場合はリフレーム必須。** トピック自体を変更するか、切り口を変えて抑制リスクを3以下に下げる:
- NG: 「宗教法人課税」→ OK: 「知らなかった！日本の非課税制度」（仕組み解説に転換）
- NG: 「創価学会の影響力」→ OK: 「連立政権が変わると何が変わる？」（政策変化に転換）
- NG: 「移民問題」→ OK: 「人手不足ニッポン、解決策は？」（社会課題に転換）
- 原則: 「[特定団体/人物] + [批判的論点]」→「[普遍的な仕組み] + [意外な事実]」

例:
- 「チームみらいの躍進」→ 対象1 + 感情1 + 議論3 = **5点（不可）**
- 「社会保険料、実は年間○○万円」→ 対象5 + 感情3 + 議論3 = **11点** / 抑制1 → **可**
- 「宗教法人課税」→ 対象3 + 感情5 + 議論5 = **13点** / 抑制5 → **リフレーム必須**

## Step 1: Web調査

WebSearch で関連情報を収集する:

- 日本語ニュース記事を3本以上検索（年号を含めて検索）
- 一次情報源（公式発表、統計データ等）を優先する
- 矛盾する情報がある場合は複数ソースで裏付けをとる

```
WebSearch: "<トピック> 2025" / "<トピック> 最新"
```

## Step 2: 情報整理・ファクトチェック

収集した情報を整理:

- 事実と意見を明確に分離する
- 数字・日付・固有名詞は元ソースと照合する
- 偏向・扇動的な表現を含まないよう注意する
- 出典URLを記録する

## Step 3: 調査メモ作成

連番を確認し、`contes/NNN_slug.research.md` に出力する:

```bash
ls contes/   # 既存の連番を確認 → 次の番号を使う
# 例: 001_team-mirai-rise.md が最後なら → 002_next-topic.research.md
```

以下のフォーマットで調査メモを作成する:

```markdown
# <トピック名>

## 概要
<トピックの簡潔な説明（2〜3文）>

## 主要な事実
- <事実1（数字・日付を含める）>
- <事実2>
- <事実3>
...

## 背景・文脈
<トピックの背景情報、なぜ今重要なのか>

## 多角的な視点
- **賛成/肯定的な見方**: ...
- **反対/批判的な見方**: ...
- **中立的な分析**: ...

## 動画の切り口候補
- <切り口1: フックになりそうな事実や疑問>
- <切り口2>
- <切り口3>

## 出典
- [記事タイトル1](URL1)
- [記事タイトル2](URL2)
- [記事タイトル3](URL3)
```

### 調査メモのポイント

- **数字を集める**: 具体的な数値は動画の説得力とフックに直結する
- **「なぜ？」を掘り下げる**: 視聴者の疑問に答える材料を揃える
- **切り口候補**: コンテ作成時にフックとして使える事実・疑問を提案する

## Step 4: ユーザーにレビュー依頼

調査メモをユーザーに提示し、以下を確認してもらう:

- 事実関係に誤りがないか
- 取り上げるべき視点が抜けていないか
- 動画の切り口はどれが良いか

レビュー完了後、`/conte contes/NNN_slug.research.md` でコンテ作成に進む。
